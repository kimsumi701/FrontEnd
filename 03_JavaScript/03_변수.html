추상화

  • 추상이란 여러가지 사물이나 개념에서 공통되는 특성이나 속성따위를 추출하여 파악하는 작용을 한다.
  • 구체적인 정보는 숨기고 꼭 필요한 핵심만을 설명하는 것을 추상화라고 한다. Abstraction
  • 예) 책 제목, 영화의 줄거리 → 세부적인 스토리나 전개, 배경은 숨기고 핵심만을 말해줌. 추상화의 결과라고 볼 수 있음.
  • 추상화를 위해서는
    목적을 명확하게 할 것
    불필요한 것은 숨길 것
    핵심만을 드러낼 것

변수

  • 데이터를 담아두는 상자의 역할
  • 변하는 수를 담을 수 있다고 해서 변수라고 한다.
  • 영어로는 Variable라고 한다.
  • 하나의 변수에는 하나의 데이터만 담을 수 있다.
  • 새로운 값이 저장되기 위해서는 원래 저장되어 있던 값은 버려지게 된다.
  • 변수를 만드는 방법(변수를 선언하는 방법)은 다음과 같다
      - var 변수이름 = 변수에 저장할 내용
      - let 변수이름 = 변수에 저장할 내용
        ※ 통상적으로 let을 많이 사용한다.
      - const 변수이름 = 변수에 저장할 내용 (이 경우 상수의 저장에 사용된다.)
        ※ 상수는 처음 선언할 때 값을 무조건 지정해줘야 한다.
        ※ 상수는 변하지 않는 고정값을 나타낸다.
        ※ 상수의 이름은 대문자로 선언한다.(오류를 일으키는 규칙은 아닌데, 강제성은 없으나 강제적인 약속이라고 이해하면 편하다. 개발자들끼리의 관례? 약속?이므로 지켜야한다.)
  • 변수 선언 시 값을 지정하지 않아도 된다.
  • 변수의 이름에는 절대로 따옴표를 사용하면 안된다. 따옴표를 사용한다는 것은 문자형 자료를 사용한다는 의미이기 때문에 결과값이 완전히 달라지게 된다.

변수 이름 만드는 규칙(안지키면 오류가 발생된다)

  • 시작은 반드시 영문 알파뱃 언더바(_), 달러기호($)로만 시작해야 한다.
    두번째 글자부터는 숫자도 사용 가능하다.
  • 대소문자는 서로 다른 대상으로 인식 된다.
  • 예약어(키워드, JavaScript가 먼저 찜뽕해놓고 의미를 부여한 단어들..)는 사용 불가능하다.
  • 그 외 지키면 좋은 규칙들
      - 의미없는 이름 사용하지 않기
      - 너무 추상적인 이름은 좋지 않다.
      - 모든 변수의 이름은 'camelCase', 'snake_case', 파스칼(NameRule)을 사용하여 작성하면 좋다.
