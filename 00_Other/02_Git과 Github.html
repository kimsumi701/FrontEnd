Git

  • 깃허브를 사용하는 목적
    1. 내 포트폴리오를 저장하는 공간
    2. 소스코드를 공유하는 공간
    3. 프로젝트 협업을 위한 공간
  • 모든 소스코드들을 컴퓨터 또는 USB에 저장하기에는 어느순간에 용량의 한계를 느끼게 된다.
  • 혹은 보관하고 있던 USB를 분실하거나, 컴퓨터에 이상이 생겨 포멧을 하게 된다면 ???
  • 이런 상황들에 있어서 공간이나 장소에 제약받지 않고 어디서나 사용할 수 이는 것이 바로 깃허브(Github)이다.
  • 프론트엔드 / 백엔드를 막라하고 많이 사용되어지는 개발자의 필수 항목이라고 할 수 있다.


Git / Github

  1. 구글에서 Github를 검색, 회원가입을 실시한다.
  2. 회원가입 후 상단 메뉴에서 레퍼지토리를 선택하여, 초록색 New 박스를 클릭, 레퍼지토리를 생성한다.
  3. 이 때 레퍼지토리의 이름과 공개범위를 설정해주어야 한다. 공개범위는 통상 Public으로 하면 된다.
  4. 레퍼지토리가 생성되고 나면 git git git git git git git으로 이루어진 영어문장들이 나타나는데 일단은 거기서 대기시키고,
  5. 다시 구글로 돌아와서 이번에는 Git을 검색한다.
  6. 가장 최상단의 Git사이트로 들어가서 다운로드 선택 후, 자신의 운영체제에 맞는 설치파일로 다운로드 해준다.
  7. 설치에 대하여 별 특이사항이 없으면 그냥 쭉 Next 클릭해서 설치해주면 된다.
  8. 설치가 끝나고 윈도우 버튼, 시작프로그램목록에서 Git이라는 폴더가 생겨있고, 그 안에 GitBash라는 것이 있다.
  9. GitBash를 실행하면, 명령프롬포트와 비슷한 창이 뜨면서 입력할 수 있는 공간이 생긴다.
  10. GitBash에서 입력하는 방법
    1. git config --global user.name "이름" -> "" 큰따음표 안의 내용 빼고 토씨하나 틀리지 말고 입력할 것.
    2. git config --global user.email "Github 가입시 입력한 이메일주소"
    3. git config --list 라고 입력해준 다음,
    4. user.name과 user.email이라는 항목에 내가 입력한 것들이 뜨면 성공한 것.
  11. 앞서 GitBash에서 설정을 마쳤으면 VSCode로 돌아와서 내가 만든 것들을 커밋해주면 된다.
  12. 이때 폴더열기가 되어 있는 상태여야한다. 폴더가 공유폴더같이 네트워크상에 공유되는 폴더는 사용이 불가능하다.
  13. 폴더 열기 하는 방법은 cmd를 통한 방법과, VSCode에서 직접 열어주는 방법이 있다.
  14. 이 때 폴더열기가 되어있으면, 그 안에 있는 내용 전체가 커밋대상에 포함되어질 수 있다.
  15. VSCode에서 Github로 커밋하는 방법
    1. VSCode 상단 메뉴에 터미널 -> 새 터미널
    2. VSCode 하단에 터미널 창이 띄워지고, 사용할 터미널의 종류를 GitBash로 바꾸어줘야 한다.
    3. 보통은 PowerShell이라는 터미널이 사용되고 있을텐데, 터미널 창의 우측 상단에 보면 +v 표시를 누르면 GitBash가 있다.
    4. GitBash 선택 후 입력창이 나타는 것을 확인 한 후 다음 명령들을 순차적으로 입력하면 된다.
    5. ------------(터미널 입력방법)--------------
    6. git init : 초기화 하는 명령. 최초에 한 번 입력하면 된다. 폴더가 바뀌거나 했을 때...
    7. git add . :
      새로운 파일들을 업로드 하겠다는 명령. 내가 원하는 파일 하나만 올리려면 git add 파일이름.확장자 로 입력하면 된다. . 이라는 것은 모든 파일을 다 올리겠다는 의미
    8. git stauts : 새롭게 커밋할 수 있는 파일들이 어떤것들이 있는지 보여주는 명령
    9. git commit -m "커밋에 관한 메세지" :
      커밋에 관한 메세지를 입력하는 명령. 히스토리 남기기.
      이때 "" 큰따음표 안에 들어가는 내용은 영문으로 내가 남길 내용들을 남기면 된다.
    10. git remote add origin https://~~~~~~~~~~.git :
      처음 회원가입하고, 깃허브에서 레퍼지토리 만들면 보이는 창에 사이트주소 같은것이 길게 보이는 것이 있는데, 그대로 복사해서 붙여넣기로 입력하면 된다.
      이 작업은 지금 내가 사용하는 VSCode와 깃허브의 레퍼지토리를 연결해주는 작업이다.
    11. git remote -v : 연결이 잘되었는지 확인해주는 명령, 입력한 주소와 같은 주소가 두줄 뜨면 성공한 것
    12. git push origin master : master 브랜치에 푸쉬하는 명령
    13. master > master 이라는 문구가 마지막에 보이면 성공
    14. Github에 레퍼지토리에 들어가서 새로고침 해보면 정상적으로 레파지토리가 생성되었음을 확인 할 수 있다.
  16. 수정사항에 대해서 다시 커밋하는 방법
    1. 코드상에 수정 사항이 생기면 연결된 GitBash가 자동으로 수정사항을 캐치하게 된다.
    2. 다시 VSCode 상단메뉴에 터미널-> 새로운 터미널에서 GitBash를 선택한다.
    3. git add . : 추가하는 명령을 입력하면 변동사항이 생긴 파일에 대해서 업로드 준비를 한다.
    4. git status : 어떤 파일들에 변동이 생겼는지를 보여준다.
    5. git commit -m "커밋 메세지 입력" : 히스토리에 커밋할 메세지를 입력한다. 영어만.
    6. git push origin master : master 브랜치에 푸쉬해준다.
    7. master > master 이라는 문구가 마지막에 보이면 성공
    8. Github에 레퍼지토리에 들어가서 새로고침 해보면 정상적으로 레파지토리가 생성되었음을 확인 할 수 있다.

Github로 협업하는 방법

    작업중인 레파지토리의 소스코드 다운받는 방법
  1. 협업을 하기 위한 레퍼지토리의 주소가 필요하므로 대상 레파지토리에 가서 초록색 CODE 버튼을 눌러서 주소를 복사해준다.
  2. 명령프롬포트(cmd)로 돌아와서 우리가 다운로드 폴더까지 cd 명령어를 이용해서 이동해준다.
  3. 이동이 완료되었다면 git clone 복사해온 주소 의 양식으로 입력한다.
  4. 입력 후 엔터를 치고, 폴더에 이동해보면 레파지토리가 통쨰로 다운로드 되어있다.
    위에서 다운받은 파일들을 작업한 후 커민하는 방법
  1. 작업을 마치고 결과물이 나오면 커밋을 해야한다.
  2. 앞서 언급한 커밋방법과 거의 동일한데 뒷부분이 조금 다르다.
  3. VSCode 상단메뉴 터미널 -> 새 터미널 -> GitBash -> 입력창이 뜨는 것을 확인 후 진행
  4. git add . : 새로운 추가요소 업로드
  5. git status : 업로드할 파일, 변동요소가 생긴 파일 확인
  6. git commit -m "커밋 메세지 입력" : 커밋 메세지 입력, 영어만 가능
  7. git push origin master : 바로 master 브랜치에 푸쉬하면 안됌.
  8. git checkout -b branchname : 개발용 새 브랜치로 들어가는 명령어
    위의 branchname에 새로 생성할 브랜치의 이름을 입력하면 된다.
  9. 위 명령을 입력하고 나면 명령줄의 제일 끝에 괄호아넹 글자가 master에서 내가 입력한 이름으로 바뀌어있는 것이 보일것이다.
  10. 브랜치의 이름이 나타나는 것인데 바뀐것을 확인 하고,
  11. git push origin branchname : 새로 생성한 브랜치로 변동사항을 푸시하겠다는 명령
    위의 branchname에 앞서 입력한 브랜치의 이름이 들어가면 된다.
    커밋 후에 Github에 와서 작업해야 할 것들
  1. Github 레파지토리에 다시 들어가보면 뭔가 노란색 박스가 떠있고, 새로운 브랜치에 글이 올라가있다는게 뜨게된다.
  2. 그 다음 compair and pullrequest 버튼을 클릭해서 들어가면 입력할 수 있는 페이지가 뜨게 된다.
  3. 거기에 메세지를 입력한 후에 초록색 박스 creat pullrequest 버튼을 눌러서 전송을 하면 된다.
  4. 이 작업 후에 마스터브랜치 관리자가, 승인을 하게 되면 최종적으로 마스터 브랜치에 나의 코드가 올라가게 되는 순서이다.
    Github를 이용한 협업은 이러한 순환의 반복이 이루어진다.
  • 내가 수정한 것을 서브 브랜치에 올리고,
  • 그 수정된 부분을 누군가가 받아서 동기화 시키고,
  • 어느 순간 마스터 브랜치에 업로드가 되고,
  • 나도 다시 마스터 브랜치를 다운받아서 동기화 시키고,
  • 다시 나의 수정본을 서브 브랜치에 업로드하고,
  • 무한 반복이 이루어진다.
